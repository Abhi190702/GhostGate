# Day 1 Checklist

Day 1 is about creating a clean project foundation.

## Repository

- [ ] Repository created
- [ ] README added
- [ ] License added
- [ ] `.gitignore` added
- [ ] Folder structure added

## Documentation

- [ ] Architecture documented
- [ ] Roadmap documented
- [ ] Learning notes started
- [ ] Ethical use policy added
- [ ] Folder READMEs added

## Safety

- [ ] No private keys committed
- [ ] No real WireGuard config committed
- [ ] No `.env` files committed
- [ ] No passwords or API tokens committed

## First Commit

Suggested commit message:

```bash
git commit -m "Initialize GhostGate project structure and documentation"
```

## Day 1 Finish Line

By the end of Day 1, the repository should clearly explain:

- what GhostGate is
- why it exists
- how the lab will be built
- what the first technical milestone is
- what is allowed and not allowed ethically

Day 2 starts the real lab work: creating the Router VM and Client VM.
